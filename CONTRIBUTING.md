# Contributing to Thesis AI

Thank you for your interest in contributing to **Thesis AI**! We welcome contributions of all kinds.

## 🚀 Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create a branch** for your feature or fix
4. **Make your changes**
5. **Test** by opening `index.html` in a browser
6. **Commit** with a descriptive message
7. **Push** to your fork
8. **Open a Pull Request**

## 📋 Guidelines

### Code Style
- This is a single-file app — keep it that way for simplicity
- Use consistent formatting (2-space indentation for JSX)
- Comment complex logic sections
- Keep variable names descriptive

### Commit Messages
Follow [Conventional Commits](https://www.conventionalcommits.org/):
```
feat: add new research field option
fix: resolve quality score parsing error
docs: update deployment instructions
style: improve mobile responsive layout
refactor: extract API call helper
```

### What We're Looking For
- **New research fields** or journal format support
- **UI/UX improvements** (accessibility, responsive design, animations)
- **Pipeline enhancements** (better prompts, new stages, citation formatting)
- **Export formats** (PDF, LaTeX, DOCX)
- **Bug fixes** and error handling improvements
- **Documentation** improvements

### What to Avoid
- Adding build tools or complex dependencies (the zero-build approach is intentional)
- Breaking the single-file architecture without discussion
- Changes that require API keys to be hardcoded

## 🐛 Reporting Bugs

Open an issue with:
- Browser and OS version
- Steps to reproduce
- Expected vs actual behavior
- Console error logs (if any)

## 💡 Feature Requests

Open an issue with the `enhancement` label describing:
- The problem you're trying to solve
- Your proposed solution
- Any alternatives you've considered
